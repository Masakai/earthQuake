## リリース内容

### Added
- `jma_intensity_web.py`: 読み取り専用 API `GET /api/events` を追加（統合システム fujimidai-observatory 連携）
  - トリガ履歴 `logs/trigger_log.jsonl` を HTTP 経由・読み取り専用で取得するエンドポイント
  - クエリパラメータ（すべて任意）: `date`（YYYY-MM-DD）/ `from` / `to`（期間）/ `limit`（新しい順、デフォルト 1000、0〜10000）/ `min_scale`（震度順序で比較、それ以上のみ）
  - ヘルパー `_read_trigger_events()` を新設。震度は `_SCALE_ORDER`（"5弱"/"5強" 等を正しく扱う）で比較
  - 正常レスポンス形式: `{"count": N, "events": [...]}`（既存キー date/ts/I/scale/ratio を保持、新しい順）

  - I/O・デコード障害は握りつぶさず伝播（障害をイベント0件に化けさせない方針）
- `test_api_events.py`: 上記 API のテスト 23 件を追加（正常系・境界・異常系 422・UTF-8 多バイト・I/O エラー伝播）

## 統合側（fujimidai-observatory）への申し送り（入力契約の追加）

reviewer 申し送りに基づき、統合側へ伝達すること:

- **入力境界が 422 になる**:
  - 不正な `min_scale`（"0".."4","5弱","5強","6弱","6強","7" 以外の値、例 "5" や "X"）→ HTTP 422 `{"error": ...}`
  - `limit` が 0〜10000 の範囲外（負値・10001 以上）→ HTTP 422
- **200 時の正常レスポンス形式は不変**: `{"count": N, "events": [...]}`。`events` 各要素は既存キー（date/ts/I/scale/ratio 等）を保持し、新しい順（date,ts 降順）。正常系のパースには影響しない。
- 統合側 config: `base_url: http://10.0.1.28:8080`, `endpoint: /api/events`。

## セキュリティ / インフラ確認事項

- 本エンドポイントは**無認証**。サーバーは `--web-bind` デフォルト `0.0.0.0` で全インターフェースにバインドする。
- `10.0.1.28:8080` が **LAN 外へ露出していないこと（非露出）をユーザーが確認済み（2026-06-15）**。
- パストラバーサル・XSS・コマンドインジェクション・認証情報漏洩なし（reviewer セキュリティ検査済み）。読み取り先は module 定数 `_TRIGGER_LOG_PATH` 固定でクエリがパスに混入しない。
