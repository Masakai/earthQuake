## 現状の名称の混在（前提の整理）

調査の結果、3つの異なる名称が並存していることを確認した。

| 場所 | 現在の名称 |
|---|---|
| GitHubリポジトリ名 | `earthQuake`（`Masakai/earthQuake`） |
| README.md 1行目 | `# rs4d-jma-intensity` |
| GitHub Pagesタイトル | `rs4d-jma-intensity — Raspberry Shake リアルタイム震度モニター` |

この混在自体が「初見で何のプロジェクトか伝わりにくい」一因になっている可能性がある。以下の提案はこの混在を解消する前提で書くが、**どの名称を正式名称にするかはユーザー判断**とし、本ドキュメントでは変更を強制しない。

## A. プロジェクトの正式な説明文（候補）

いずれも「Raspberry Shakeのモニタ」という印象を避け、月次レポート・HVSR・地盤解析・継続観測という要素を含めることを意図した候補である。

### 日本語版

| 候補 | 意図・根拠 |
|---|---|
| 個人地震観測所 | 「モニタ」ではなく「観測所」とすることで、リアルタイム表示だけでなく月次レポート・地盤解析までを含む継続的な営みであることを示す。平易で一般読者にも伝わる。 |
| 個人観測点による地震・地盤モニタリング基盤 | 機能の広さ（震度算出・地盤解析・レポーティング）を説明的に示す。やや長いがREADME本文向き。 |
| 市民地震学プラットフォーム | ユーザー自身が指摘した「Citizen Seismology」の直訳。学術的な文脈を持ち込みたい場合に有効だが、「プラットフォーム」という語がやや大仰に響く可能性がある。 |

### 英語版

| 候補 | 意図・根拠 |
|---|---|
| Personal Seismic Observatory | 「観測所」の直訳。個人運用であることと継続観測であることの両方を示す。 |
| Citizen Seismology Toolkit | 学術分野で実際に使われる「Citizen Seismology」という語を採用しつつ、「Toolkit」で個人開発ツール群であることを示す。 |
| Raspberry Shake Real-time Seismic Observatory | ユーザー案そのまま。Raspberry Shakeという具体的ハードウェア名を残すことで検索性を確保しつつ、「Observatory」で単なるモニタ以上の機能を示唆する。 |

**推奨の方向性（根拠付き）**: どれか1つに絞るのではなく、「サブタイトル」として使う運用を提案する。理由は、`rs4d-jma-intensity`（技術名）・`earthQuake`（リポジトリ名）という既存の識別名をどちらも変えずに済み、後方互換性（既存のブックマーク・検索結果・リンク）を壊さないため。例えば README冒頭を以下のような構成にする案（本文はフェーズ6で確定）。

```
# rs4d-jma-intensity

個人地震観測所 — Raspberry Shake 4Dによるリアルタイム震度計算・地盤特性解析・継続観測レポート
```

のように、既存タイトル行は残し、直下に位置づけを示す1文を追加する形であれば破壊的変更を避けられる。

## C. GitHub Pagesトップページの再構成

現状（外部調査結果）はスクリーンショット・機能一覧・月次レポート導線が既に存在しており、ゼロから作る必要はない。「10秒以内に理解できる」ためには、以下の並び順の見直しが効果的と考えられる。

1. 冒頭に1文の価値提案（Aの候補を反映）を追加し、その直下に「ライブ画面イメージ」「月次レポート例」「GitHub」の3ボタンを横並びで配置する
2. 現行の機能一覧（10項目）の先頭に、HVSR・月次レポート自動生成・地盤特性検証を明示的な項目として追加する（現状これらが埋没している）
3. フッター付近に「関連記事」導線（Qiita・ブログ一覧へのリンク）を新設する（現状皆無）

## D. コンテンツ導線の設計

現状: GitHub ⇄ GitHub Pages ⇄ Qiita ⇄ ブログの4媒体はほぼ孤立（現状評価レポート6節）。

提案する導線:

```
ブログ記事（物語の入口）
  → 記事末尾に「実装の詳細はGitHub」「ライブ画面・月報はプロジェクトサイト」の2リンク

Qiita記事（技術解説の入口）
  → 記事末尾に「プロジェクト全体・運用実績はプロジェクトサイト」のリンク（現状GitHubのみ）

GitHub Pagesトップ（総合案内）
  → 「開発の背景（ブログ）」「技術詳細（Qiita）」「ソースコード（GitHub）」の3リンクをフッターに設置

GitHub README
  → 既存の「詳細はMANUAL.mdを参照」に加え、「開発の背景・関連記事」節を新設しQiita・ブログへリンク
```

これにより、どの媒体から訪れても他の3媒体に到達できる状態を作る。GitHub Pages・READMEの改訂はフェーズ6で実施可能。Qiita・ブログの編集は本リポジトリ外のため、実施はユーザー判断（プランとして提示のみ）。
